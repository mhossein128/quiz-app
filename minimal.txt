## 1. INTRODUCTION

### Overview
Minimal UI is a professional React Kit that comes with plenty of ready-to-use Material-UI components that will help you to build more beautiful frontend pages.

### Key Features
- Built with MUI (Material-UI) with two versions: Next.js | Vite
- Includes a complete design component Figma file to extend your project development
- The theme is ready to change to any style you want
- Professional dashboard and client templates
- Complete authentication system
- Multi-language support
- Responsive design

### Framework Support
- Next.js v14 (next-app-dir)
- React v18
- MUI v6
- Vite.js
- Create React App (CRA)

### Documentation References
- MUI Components: https://mui.com/components/
- Third-party libraries with direct links
- Example: Carousel - https://www.embla-carousel.com/

### Project Scope
This project is just the user interface, not including the backend and database.

### Configuration Setup
Generic configuration for several APIs located in global-config.ts:

```javascript
export const CONFIG: ConfigValue = {
  appName: 'Minimal UI',
  appVersion: packageJson.version,
  serverUrl: process.env.NEXT_PUBLIC_SERVER_URL ?? '',
  assetsDir: process.env.NEXT_PUBLIC_ASSETS_DIR ?? '',
  isStaticExport: JSON.parse(`${process.env.BUILD_STATIC_EXPORT}`),
  
  // Authentication
  auth: {
    method: 'jwt', // jwt | amplify | firebase | supabase | auth0
    skip: false,
    redirectPath: paths.dashboard.root,
  },
  
  // Mapbox
  mapboxApiKey: process.env.NEXT_PUBLIC_MAPBOX_API_KEY ?? '',
  
  // Firebase
  firebase: {
    apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY ?? '',
    authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN ?? '',
    projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID ?? '',

    messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID ?? '',

    measurementId: process.env.NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID ?? '',

  // AWS Amplify
  amplify: {
    userPoolId: process.env.NEXT_PUBLIC_AWS_AMPLIFY_USER_POOL_ID ?? '',
```

## 13. DEPLOYMENT & PRODUCTION

### Mock Server Setup

#### Local Development
1. Download resource from README.md or MOCK_API.md
2. Start local server in minimal-api-dev folder
3. Ensure running on correct port: http://localhost:7272
4. Update .env in your project:

```bash
VITE_SERVER_URL=http://localhost:7272
VITE_ASSET_URL=http://localhost:7272
```

#### Production Deployment
1. Push minimal-api-dev source code to GitHub
2. Deploy on Vercel
3. Update next.config.mjs:

```javascript
const nextConfig = {
  reactStrictMode: true,
  env: {
    DEV_API: 'http://localhost:7272',
    PRODUCTION_API: 'https://your-domain-api.vercel.app',
  },
};

export default nextConfig;
```

4. Update .env for production:

```bash
VITE_SERVER_URL=https://your-domain-api.vercel.app
VITE_ASSET_URL=https://your-domain-api.vercel.app
```
